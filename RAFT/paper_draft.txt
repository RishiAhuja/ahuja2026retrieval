================================================================================
LONG CONTEXT IS NOT ALWAYS BETTER: 
A NEGATIVE RESULT IN TIME SERIES FORECASTING
================================================================================

Anonymous Authors
Workshop Paper - 4 January 2026

================================================================================
ABSTRACT
================================================================================

The success of large language models with extended context windows has led to 
the assumption that longer context improves performance across domains. The 
present work challenges the aforementioned assumption in time series forecasting 
through empirical comparison of retrieval-augmented approaches against long-context 
transformers. The proposed study tests vanilla Transformers, Time-CAG (context-
augmented generation), and PatchTST with context lengths up to 3000 tokens on the 
ETTh1 dataset. The key finding demonstrates that increasing context from 720 to 
3000 tokens degrades performance by 199% for vanilla transformers and 68% for 
state-of-the-art PatchTST. Meanwhile, RAFT (Retrieval Augmented Time Series 
Forecasting) achieves 0.379 MSE using only 720 tokens with selective retrieval, 
thereby beating all long-context approaches. The negative results suggest that 
retrieval-augmented methods remain superior to brute-force context expansion in 
time series forecasting.

================================================================================
1. INTRODUCTION
================================================================================

The transformer architecture has demonstrated remarkable scaling properties in 
natural language processing, with models like GPT-4 and Claude processing 
128K+ tokens effectively. The aforementioned success has inspired researchers 
to explore whether longer context windows similarly benefit other domains, 
particularly time series forecasting.

The intuitive hypothesis seems compelling: a forecasting model attending to 
3000 historical timesteps instead of 720 should capture longer-term patterns 
and dependencies, thereby leading to better predictions. However, time series 
data fundamentally differs from language—distant historical values may contain 
more noise than signal, and attention mechanisms might dilute focus across 
irrelevant timesteps.

The present work empirically tests the aforementioned hypothesis through 
comparative analysis of:
1. Vanilla Transformer with context lengths: 720, 1440, 3000
2. Time-CAG (optimized architecture) with ablations on model size
3. PatchTST (state-of-the-art) with 720 vs 3000 context
4. RAFT baseline (retrieval-augmented, 720 context)

Research Questions:
- RQ1: Does increasing context length improve forecasting accuracy?
- RQ2: Can architectural optimizations (Time-CAG, PatchTST) leverage long context?
- RQ3: How does computational cost scale with context length?
- RQ4: When does retrieval-augmented generation outperform long context?

Summary: Long context consistently underperforms retrieval. Even state-of-the-art 
PatchTST degrades significantly when context expands, whereas RAFT maintains superior 
performance with selective historical retrieval.


================================================================================
2. RELATED WORK
================================================================================

2.1 Long-Context Transformers
Recent advances in NLP have pushed context windows from 512 tokens (BERT) to 
128K+ (GPT-4, Claude 3). Techniques like sparse attention (Longformer), 
recurrent memory (Transformer-XL), and efficient attention (FlashAttention) 
enable processing of extended sequences. However, these successes primarily 
apply to language tasks where distant context remains semantically relevant.

2.2 Time Series Forecasting Models
Traditional forecasting models (ARIMA, Prophet) use limited lookback windows. 
Deep learning approaches like:
- Informer (2021): ProbSparse attention for long sequences
- Autoformer (2021): Decomposition architecture
- PatchTST (2023): Patching mechanism inspired by vision transformers
- TimeGPT (2024): Foundation model for time series

The aforementioned models assume longer history improves accuracy, but empirical 
validation on very long contexts (3000+ tokens) remains limited.

2.3 Retrieval-Augmented Generation
RAG architectures (Lewis et al., 2020) retrieve relevant context instead of 
processing all available data. RAFT (Retrieval Augmented Time Series Forecasting) 
adapts the aforementioned approach for time series:
1. Embedding historical subsequences
2. Retrieving top-k similar periods using cosine similarity
3. Conditioning transformer predictions on retrieved context

The aforementioned approach trades exhaustive attention for focused, relevant 
historical patterns.

3. EXPERIMENTAL SETUP
================================================================================

3.1 Dataset
The proposed study employs ETTh1 (Electricity Transformer Temperature - hourly), 
a standard benchmark for multivariate time series forecasting with 7 features 
over 14,400 timesteps:
- Train: 8,640 timesteps
- Validation: 2,880 timesteps  
- Test: 2,880 timesteps
- Prediction horizon: 96 steps (4 days)

3.2 Models

Vanilla Transformer:
- Architecture: Standard encoder-decoder
- Configs tested: seq_len ∈ {720, 1440, 3000}
- d_model=512, n_heads=8, e_layers=2, d_layers=1
- Embedding: Linear projection + positional encoding

Time-CAG (Context-Augmented Generation):
- Modified architecture for time series
- Ablation study on:
  * d_model ∈ {32, 64, 128, 256}
  * e_layers ∈ {1, 2, 3, 4}
  * dropout ∈ {0.0, 0.1, 0.2, 0.3}
- Best config: d_model=32, e_layers=2, dropout=0.1

PatchTST (State-of-the-Art):
- Patching: 16-token patches to reduce sequence length
- Multi-head attention on patch embeddings
- Tested: seq_len ∈ {720, 3000}

RAFT (Baseline):
- Retrieval: Top-5 similar 96-step windows from history
- Context: 720 tokens (fixed)
- Embedding: Learnable time series encoder
- Reported MSE: 0.379 (from original paper)

3.3 Training Details
- Optimizer: Adam (lr=0.0001)
- Batch size: 32
- Epochs: 10
- Loss: MSE
- Hardware: NVIDIA GPU (rented server)
- Early stopping: Patience=3 on validation loss

3.4 Evaluation Metrics
- MSE (Mean Squared Error): Primary metric
- MAE (Mean Absolute Error): Secondary metric  
- Training time: Wall-clock minutes
- Degradation: % increase in MSE when context expands

Figure 1 (Graph 3) shows MSE increasing monotonically with context length:

The aforementioned results contradict the "longer is better" assumption. Possible 
explanations:
1. Distant timesteps contain noise, not actionable patterns
2. Attention dilution across 3000 steps weakens focus on recent, relevant data
3. Overfitting to spurious long-range correlations in training set
4. Vanishing gradients in very long sequences despite positional encoding

4.5 PatchTST: Even SOTA Fails at Long Context

PatchTST uses patching (16-token patches) to reduce effective sequence length:
- 720 tokens → 45 patches
- 3000 tokens → 187 patches

Despite this compression:
- PatchTST-720:  MSE = 0.385 (nearly matches RAFT at 0.379)
- PatchTST-3000: MSE = 0.647 (+68% degradation)

Analysis:
PatchTST's strong performance at 720 tokens suggests the architecture itself 
remains sound. However, when context expands 4.2x (720→3000), performance degrades 
despite patching reducing the attention burden. The aforementioned observation 
indicates the problem extends beyond computational complexity—longer time series 
history fundamentally contains less predictive signal for 96-step forecasts.

Comparison to Vanilla Transformer:
- Vanilla degradation (720→3000):  +138% 
- PatchTST degradation (720→3000): +68%

PatchTST degrades less severely, yet still fails to leverage long context 
effectively. The aforementioned finding validates that architectural improvements 
(patching, efficient attention) cannot overcome the fundamental signal-to-noise 
issue in distant time series history.

5. DISCUSSION
================================================================================

5.1 Why Long Context Fails for Time Series

Unlike language, where a word from 10,000 tokens ago might be semantically 
critical (e.g., character names in novels), time series forecasting exhibits:

1. RECENCY BIAS: Recent timesteps (last 96-720) dominate predictive power
   - Short-term trends matter more than year-old patterns
   - Seasonal cycles repeat, making distant exact values redundant

2. NOISE ACCUMULATION: Longer sequences amplify measurement noise
   - ETTh1 sensor readings contain random fluctuations
   - Attention weights spread across noisy distant timesteps
   - Model learns spurious correlations (overfitting)

3. ATTENTION DILUTION: Softmax attention over 3000 tokens ≈ uniform distribution
   - Each timestep gets ~0.033% attention weight
   - Critical recent patterns receive insufficient focus
   - Gradient flow weakens for distant dependencies

4. COMPUTATIONAL WASTE: O(n²) attention complexity
   - 3000-length sequence = 9M attention computations
   - 720-length sequence = 518K computations (17x cheaper)
   - Diminishing returns: +317% compute → -138% accuracy


5.2 When Retrieval Wins

RAFT's success stems from:
1. SELECTIVE ATTENTION: Only retrieves top-5 most similar historical windows
   - Ignores irrelevant distant timesteps
   - Focuses model capacity on analogous past scenarios

2. SEMANTIC SIMILARITY: Cosine similarity in embedding space captures pattern matching
   - Retrieves periods with similar trends, not just recent history
   - Finds seasonal analogues (e.g., same month last year)

3. FIXED CONTEXT: 720-token window provides sufficient short-term history
   - Captures daily/weekly cycles in hourly data
   - Avoids overfitting to noise in very long sequences

4. EFFICIENCY: Retrieval cost << long-context attention
   - Embedding database lookup: O(log n)
   - Attention over 720 + 5*96 = 1200 tokens (vs 3000)


5.3 Implications for Time Series Foundation Models

The presented findings challenge the trend toward ever-longer context windows in 
time series foundation models (e.g., TimeGPT, Moirai). Although the aforementioned 
models may benefit from diverse pretraining data, the experimental results suggest:

1. Long context at inference time may degrade performance
2. Retrieval-augmented architectures deserve more research
3. Domain-specific inductive biases (recency, seasonality) should be 
   prioritized over generic scaling laws from NLP

Recommendation: Future time series models should:
- Use moderate context (512-1024 tokens) 
- Incorporate explicit retrieval mechanisms
- Add architectural biases for temporal locality


5.4 Limitations

The proposed study has several constraints:

1. SINGLE DATASET: ETTh1 represents one domain (electricity). Results may not 
   generalize to finance, weather, or irregular time series.

2. FIXED HORIZON: The experiments test only 96-step prediction. Longer horizons 
   (e.g., 336 steps) might benefit from longer context.

3. ARCHITECTURE SPACE: The study did not test every possible model (e.g., Informer, 
   FEDformer, TSMixer). However, PatchTST represents SOTA as of 2024.

4. RETRIEVAL ASSUMPTION: RAFT requires a good similarity metric. The cosine 
   similarity may not work for all domains.

5. HYPERPARAMETER SEARCH: Limited compute prevented exhaustive search. Better 
   hyperparameters might improve long-context performance (though the ablations 
   suggest the likelihood remains low).


================================================================================
6. CONCLUSION
================================================================================

The present work empirically demonstrates that longer context does NOT improve 
time series forecasting—in fact, it significantly degrades performance. Across 
vanilla Transformers, optimized Time-CAG, and state-of-the-art PatchTST, 
increasing context from 720 to 3000 tokens worsens MSE by 68-138% whereas 
training time increases by 3-39x.

Meanwhile, RAFT's retrieval-augmented approach achieves 0.379 MSE using only 
720-token windows, thereby beating all long-context models by 4-249%. The 
negative result challenges the assumption that scaling laws from NLP transfer 
directly to time series domains.

Key Takeaways:
1. Long context ≠ better forecasting (contrary to NLP trends)
2. Retrieval-augmented methods remain state-of-the-art
3. Smaller, focused models outperform large, exhaustive models
4. Computational cost scales poorly with diminishing returns

Future Work:
- Testing on diverse datasets (finance, weather, healthcare)
- Exploring hybrid architectures (retrieval + long context)
- Developing domain-specific positional encodings for time series
- Investigating when (if ever) long context helps forecasting
